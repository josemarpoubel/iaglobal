In iaglobal, agents cannot call LLM directly...

1. **BanditPolicy as sole gatekeeper** —  All LLM access passes through:
- ε-greedy model selection
- Circuit breaker (does not call provider with recent failures)
- Fallback chain (if cloud fails → Ollama local)
- CreditAssignmentEngine (register success/failure)
- Reward assignment (update BanditPolicy)

2. **Agents as specialized wrappers** — Each agent has:
- Domain-specific prompt template
- Output validation (schema, quality)
- Retry logic with backoff
- Immune memory hooks (EvoAgent)

3. **Critic as validator** — The CriticAgent:
- Validates outputs from other agents
- Decides if rework is needed
- Approves before proceeding in the pipeline

## 🫀 The SocialRegistry — Horizontal Agent Cooperation (July 2026)

Before July 2026, agents could only communicate **vertically** (Agent → Critic → Bandit → Provider). There was no protocol for one agent to discover another agent's capabilities and delegate work directly.

The `SocialRegistry` solved this by introducing a **hormonal advertisement system**:

```
┌─────────────────────────────────────────────────────────────────┐
│  SocialRegistry (singleton)                                     │
│                                                                 │
│  publish(adv)   → upsert skills + load_factor                   │
│  heartbeat(id)  → update last_seen (TTL=120s)                   │
│  withdraw(id)   → remove agent (voluntary apoptosis)            │
│  query(domain)  → live list sorted by proficiency descending    │
│  get(id)        → Advertisement or None (stale returns None)    │
│                                                                 │
│  AcetylcholineBus channels:                                     │
│    social.agent.advertise  → _on_advertise  → publish()         │
│    social.agent.heartbeat  → _on_heartbeat  → heartbeat()       │
│    social.agent.withdraw   → _on_withdraw   → withdraw()        │
└─────────────────────────────────────────────────────────────────┘
```

**Before** (vertical only):
```
Agent A ──request──→ Critic ──evaluate──→ Bandit ──select──→ Provider
```

**After** (vertical + horizontal):
```
                        ┌─ SocialRegistry ─────────────────────┐
                        │  agent-b: code(0.92)  load=0.3       │
                        │  agent-c: test(0.85)  load=0.1       │
                        └──────────────────────────────────────┘
                               ↑ heartbeat / advertise
                               ↑
```

## 🖥️ System Administration — Preventing RAM Exhaustion

Large-scale pipeline execution (81+ nodes running concurrently) can exhaust available RAM. iaglobal uses swap space as a metabolic buffer — when physical memory fills, the kernel pages inactive data to disk instead of OOM-killing processes.

### Recommended Swap Size

| Scenario | Physical RAM | Recommended Swap |
|----------|-------------|-----------------|
| Development / single pipeline | ≤ 16 GiB | 2× RAM (32 GiB) |
| Multi-session / agent colonies | 16–32 GiB | 1.5× RAM (24–48 GiB) |
| Production / continuous evolution | ≥ 32 GiB | 1× RAM minimum, 48 GiB recommended |

> **Rule of thumb**: iaglobal with 81 pipeline nodes + Obsidian vault + OmniMind state can spike to 14+ GiB under load. Configure swap = physical RAM or higher.

### Setting Up Swap (Linux)

#### Option A — Swap Partition (if available)

```bash
# Check current swap
cat /proc/swaps

# If you have a swap partition large enough, enable it:
sudo swapon /dev/<your-swap-partition>
```

#### Option B — Swap File (recommended for most setups)

```bash
# 1. Allocate a 12 GiB swap file (adjust size as needed)
sudo fallocate -l 12G /swapfile

# 2. Secure it (root-only access)
sudo chmod 600 /swapfile

# 3. Format as swap
sudo mkswap /swapfile

# 4. Enable immediately
sudo swapon /swapfile

# 5. Verify
cat /proc/swaps
free -h
```

#### Persist Across Reboot

Add the swap file to `/etc/fstab`:

```bash
echo "/swapfile none swap sw 0 0" | sudo tee -a /etc/fstab
```

#### Verify Final State

```bash
$ free -h
               total       used       free
Mem:            23Gi       14Gi      8.8Gi
Swap:           24Gi      4.8Gi       19Gi   ✓
```

### Memory Pressure Monitoring

```bash
# Watch swap usage in real time
watch -n 2 free -h

# Detect RAM pressure events
grep -i "oom\|out of memory" /var/log/syslog

# iaglobal's own mitochondrial probe logs hypoxia events:
#   [MITO-PROBE] ⚠️ GRADIENTE EM COLAPSO: 60ms de lag (threshold=50ms)
# When you see this, the event loop is struggling — add more swap.
```

### ⚠️ Important Notes

- **Swap is not a substitute for RAM** — it is a safety net. If iaglobal consistently uses >80% of swap, add physical RAM.
- **SSD wear**: Heavy swapping on consumer SSDs can reduce lifespan. Use a dedicated swap partition on a separate NVMe or SSD for production.
- **Swap file vs partition**: Files are easier to resize (`fallocate` + `mkswap` + `swapon`). Partitions are slightly faster.
- **Docker / containers**: If running iaglobal inside Docker, configure swap via `--memory-swap` flag (`--memory-swap=48g --memory=24g`).

## 🚦 Traffic Light Architecture — Release Chain

O sistema de semáforos (TokenBucket + asyncio.Semaphore) protege o modelo local
(qwen2.5:0.5b, glm4, lfm) contra thrashing de CPU. A **correção crítica** de
Julho/2026 fechou o vazamento de `current_concurrent` que travava o tier JUIZ
após 2 chamadas.

### Cadeia de Acquire/Release (corrigida)

```
acquire_model(model, node_id)
  ├─ LocalModelGate.try_acquire → TokenBucket.acquire()
  │    ├─ current_concurrent += 1
  │    └─ tokens -= 1
  └─ asyncio.Semaphore.acquire()

generate() → finally:
  ├─ release_model(model) → asyncio.Semaphore.release()
  └─ LocalModelGate.release(node_id) → TokenBucket.release()
       └─ current_concurrent -= 1    ← NOVO: fecha o ciclo
```

### Fallback seguro

Quando `acquire_model` retorna `False` com token já consumido, o código libera
o bucket **antes** de tentar o fallback — evitando leak duplo.

### Arquivos tocados

| Arquivo | Mudança |
|---------|---------|
| `execution/token_bucket.py` | `LocalModelGate.release()` — novo método |
| `graphs/bandit.py` | `generate()` finally + fallback — release simétrico |
