ESPECIFICACOES DO GENESIS
--------------------------------------------------------------------------------

1. Toda identidade de agente e gerada a partir deste genesis_id via:
   agent_id = sha3_512(genesis_id + ":" + role + ":" + version)

2. Toda chamada cross-agent exige validacao de linhagem:
   if caller.genesis != callee.genesis: REJECT

3. Toda evolucao cria nova geracao mantendo genesis_id (mesma familia):
   generation = sha3_512(genesis_id + ":gen:" + parent_agent_id)

4. Hash de linhagem completo:
   lineage_hash = sha3_512(genesis_id + parent_id + child_id + timestamp + payload)

================================================================================
LINEAGE_MANIFEST (gerado no DOCUMENTATION e atualizado por toda a pipeline)
================================================================================

Cada agente, ao ser instanciado, registra:

genesis_id: GENESIS_7F3A9B2C8D1E4F6A5B9E2D7C1A8F3B6E9D4C2A7B5E8F1A3C6D9B2E5A8C1F4D7B
agent_id: sha3_512(genesis_id + ":" + role + ":" + version)
parent_id: id do agente que invocou este (ou genesis_id se for raiz)
generation: numero da geracao dentro da familia
lineage_hash: sha3_512(genesis_id + parent_id + agent_id + timestamp + payload_hashed)
criado_em: ISO8601 timestamp
criado_por: id do caller
hash_prompt: sha3_512 do prompt usado (pra auditoria)

Validacao cross-agent (executada antes de qualquer chamada):
  if caller.genesis != self.genesis:
    raise LineageViolationError(f"Cross-family call blocked: {caller.genesis} -> {self.genesis}")
