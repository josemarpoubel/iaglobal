# 🧬 YaCy + iaglobal — Busca Distribuída e Soberana

## Filosofia

YaCy é um buscador open-source peer-to-peer (P2P) onde cada nó da rede
crawleia uma parte da web e compartilha o índice com os demais — sem
servidor central.

A integração com iaglobal transforma cada nó iaglobal em um **nó YaCy**
que participa da rede global de busca, ao mesmo tempo que usa o sistema
imunológico e evolutivo do iaglobal para auto-gerenciar o crawl,
filtrar resultados tóxicos e otimizar o índice por demanda.

---

## Estado Atual

- iaglobal depende de 4 fontes de busca externas (DuckDuckGo, You.com,
  Bing, Grokipedia) — todas com rate limits, dependência de API keys
  ou bloqueios
- Não há índice local próprio — cada consulta é uma requisição a
  terceiros
- `no_search.py` orquestra fontes em paralelo mas não possui fonte
  nativa

---

## Estado Desejado

- Cada nó iaglobal roda um **YaCy peer leve** que contribui com o
  índice global e consulta a rede P2P
- Crawl é distribuído: cada nó crawleia domínios específicos e
  compartilha
- Busca funciona **sem API keys, sem rate limits, sem bloqueios**
- A rede YaCy + iaglobal cresce organicamente: cada novo nó fortalece
  o índice coletivo
- O sistema imunológico (MHC) do iaglobal filtra spam, malware e
  conteúdo tóxico do índice P2P
- A BanditPolicy otimiza quais domínios cada nó crawleia baseada na
  demanda real de busca

## Plano de Implantação

### Fase 1 — Peer Local (1-2 dias)

1. Instalar YaCy:

```
   docker run -d --name yacy -p 8090:8090 \
     -v yacy_data:/opt/yacy/data \
     konstin2/yacy
```

2. Verificar API:

```
   curl http://localhost:8090/yacysearch.json?query=test&maximumRecords=3
```

3. Criar `yacy_search()` em `_search_sources.py`:

```python
   def yacy_search(query: str) -> str:
       """Busca via YaCy peer local."""
       try:
           q = urllib.parse.quote(query)
           url = f"http://localhost:8090/yacysearch.json?query={q}&maximumRecords=5"
           with urllib.request.urlopen(url, timeout=10) as r:
               data = json.loads(r.read().decode("utf-8"))
           results = []
           for item in data.get("channels", [{}])[0].get("items", []):
               title = item.get("title", "")
               link = item.get("link", "")
               snippet = item.get("description", "")[:200]
               if title and link:
                   results.append(f"• {title}\n  {link}\n  {snippet}")
           return "\n\n".join(results)
       except Exception as e:
           logger.debug("[YACY] Fail: %s", e)
           return ""
```

4. Registrar em `no_search.py` como fonte com prioridade alta.

### Fase 2 — Crawl Distribuído (3-5 dias)

1. Criar `no_yacy_crawler.py` — nó que gerencia o crawler YaCy:
   - Recebe domínios para crawlear via `AcetylcholineBus`
   - Define profundidade, frequência, politeness
   - Reporta stats via `execution_metrics`

2. Conectar `BanditPolicy` ao crawler:
   - Domínios mais buscados → maior prioridade de crawl
   - Domínios com alta taxa de erro → pausa ou remove

3. Configurar `PhospholipidRegistry` para anunciar o peer na rede:
   ```
   /yacy/peers/<node_id> → {"address": "tcp://...", "index_size": N}
   ```

### Fase 3 — Imunidade ao Índice P2P (5-7 dias)

1. Adaptar `mhc_detector.py` para avaliar resultados da rede YaCy:
   - Fingerprint de resultados maliciosos
   - Score de confiança por peer
   - Peer com score baixo → ignorado

2. Criar vacinas compartilhadas no `ImmuneMemoryExchange`:
   - Padrões de spam detectados → broadcast para rede
   - URLs maliciosas → blacklist distribuída

3. `MetabolicPruner` para limpeza de índices obsoletos:
   - TTL por domínio baseado em frequência de mudança
   - Deduplicação de resultados

### Fase 4 — Lib Cliente e Incentivos (7-14 dias)

1. `pip install iaglobal-search` — lib mínima que:
   - Inicia peer YaCy leve
   - Conecta à rede iaglobal via `AcetylcholineBus`
   - Participa do crawl voluntário
   - Retorna resultados de busca

2. Protocolo de reputação:
   - Peers com alto uptime e índice grande → maior peso
   - Peers com MHC validado → status "confiável"
   - Peers maliciosos → expulsos via apoptose social

3. Dashboard da rede:
   - Quantos peers ativos
   - Tamanho total do índice
   - Consultas por segundo
   - Domínios mais crawleados

## Benefícios

### Para o Usuário
- **Busca sem censura** — nenhuma entidade controla o índice
- **Sem rate limits** — a rede é os próprios usuários
- **Sem anúncios** — não há modelo de negócio baseado em coleta de dados
- **Dados soberanos** — cada um contribui voluntariamente com o que quer
- **Offline-first** — índice local responde mesmo sem internet

### Para o iaglobal
- **Fonte de busca nativa** — sem dependência externa de APIs
- **Índice vivo** — evolui com a rede, não precisa de recrawl manual
- **Resiliência** — se um peer cai, outros têm o dado
- **Custo zero** — sem API keys, sem provedor de busca
- **Escala elástica** — mais peers = mais índice = melhores resultados

### Para a Sociedade
- **Alternativa real ao Google** — não concentrada, não corporativa
- **Preservação da web aberta** — cada link crawleado é um backup
- **Democratização do conhecimento** — qualquer um participa
- **Imune a aquisições** — não há empresa para comprar

## Comparação: Antes vs Depois

| Aspecto | Antes (só APIs) | Depois (com YaCy) |
|---------|-----------------|-------------------|
| Fontes de busca | 4 APIs externas | 4 APIs + YaCy P2P |
| Dependência | API keys, rate limits | Nenhuma |
| Índice | Zero (só consulta) | Local + distribuído |
| Cobertura | Limitada ao que as APIs indexam | Qualquer domínio crawleado por alguém |
| Latência | 1-7s por fonte | ~100ms local, ~1s P2P |
| Bloqueio | Google bloqueia scraping | Impossível bloquear todos os peers |
| Custo | Gratuito limitado | Zero |

## Potencial Evolutivo

Uma vez rodando com YaCy, o iaglobal pode evoluir para:

1. **Crawl seletivo por IVM** — cada peer crawleia domínios com alto
   IVM (produtividade × eficiência × cooperação)
2. **Ranking federado** — o rank dos resultados não é definido por um
   algoritmo central, mas pelo consenso dos peers via Proof of Index
3. **Mercado de crawl** — peers com mais recurso (CPU/rede) podem
   crawlear mais e ganhar reputação na rede
4. **Auto-cura do índice** — se um domínio morre (404), o índice é
   automaticamente removido de todos os peers via vacina
5. **Bridge para IPFS** — resultados de busca podem apontar para
   conteúdo no IPFS, não apenas HTTP
