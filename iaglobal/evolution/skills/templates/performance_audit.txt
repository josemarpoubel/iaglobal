ÁREAS DE ANÁLISE:

1. COMPLEXIDADE ALGORÍTMICA
   - Big O de funções críticas
   - Loops aninhados desnecessários?
   - Operações O(n²) ou pior?

2. BANCO DE DADOS
   - N+1 queries detectadas?
   - Índices faltando?
   - Query optimization possível?
   - Connection pooling configurado?

3. CACHING
   - Dados repetidamente buscados?
   - Cache hits/misses analisados?
   - TTL adequado?
   - Cache invalidation strategy?

4. MEMÓRIA
   - Memory leaks potenciais?
   - Estruturas de dados adequadas?
   - Large objects em memória?
   - Streaming vs buffering?

5. I/O E REDE
   - Operações síncronas bloqueantes?
   - Connection pooling para HTTP?
   - Compression habilitada?
   - CDN para estáticos?

6. CONCURRENCIA
   - Thread-safe quando necessário?
   - Race conditions possíveis?
   - Deadlocks potenciais?
   - Async/await usado corretamente?

7. BOTTLENECKS
   - Ponto mais lento do sistema?
   - Escalabilidade vertical vs horizontal?
   - Load balancing adequado?
