# ESTRUTURAS SUPORTADAS

## CRISPE
- **C**ontext: Contexto da tarefa
- **R**ole: Persona do LLM
- **I**nstruction: Instrução clara
- **S**teps: Passos para seguir
- **P**ersona: Tom e estilo
- **E**xamples: Few-shot examples

## RTF
- **R**ole: Quem o LLM deve ser
- **T**ask: O que deve fazer
- **F**ormat: Como deve formatar saída

## CO-STAR
- **C**ontext: Contexto
- **O**bjective: Objetivo
- **S**tyle: Estilo de escrita
- **T**one: Tom
- **A**udience: Público
- **R**esponse: Formato de resposta

# TÉCNICAS DE OTIMIZAÇÃO

## 1. Few-Shot Prompting
- Inclua 2-3 exemplos relevantes
- Cubra casos comuns e edge cases
- Formate exemplos consistentemente

## 2. Chain-of-Thought
- Peça para LLM "pensar passo a passo"
- Useful para problemas complexos
- Melhora raciocínio lógico

## 3. Delimitadores Claros
- Use ###, ---, """, ```
- Separe seções claramente
- Facilita parsing pelo LLM

## 4. Especificação de Formato
- JSON, XML, Markdown, etc
- Inclua schema ou template
- Reduz alucinações

## 5. Constraints Explícitas
- Word limits
- Style guides
- Do's and Don'ts

# VALIDAÇÃO
Antes de retornar:
1. ✅ Prompt é claro e específico?
2. ✅ Formato de saída está explícito?
3. ✅ Exemplos são relevantes?
4. ✅ Constraints são razoáveis?
5. ✅ Token count está dentro do limite?

# TOKEN ESTIMATION
- Use ~4 chars por token (média)
- Considere overhead do sistema
- Deixe margem para response
